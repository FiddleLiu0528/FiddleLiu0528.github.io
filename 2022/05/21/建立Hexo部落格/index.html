建立Hexo部落格

引言

本篇文章的目的為建立自己 github 自動部屬的流程。
透過這篇文章，可以在推送部落格專案至 github 的同時，自動部屬最新的部落格。

環境

建立的環境使用 windows 10
使用的文字編輯器 vs code
先安裝 node.js ，透過 node 運行封裝好的部落格工具套件 Hexo 開發自己的部落格。
透過 git 建立版本控制，可以快速管理部落格的版本。

安裝好 git 後，要先在全局設定自己的 git 資訊，這些資訊會是之後每次修改時，紀錄的作者資訊。
也可以至 “C:\Users\{user}\.gitconfig” 檔案手動調整。

使用的版控管理工具是 source tree。

部屬網站的地方為 github，所以需要先建立屬於自己的帳號。

有了以上這些環境及帳號，就可以準備打造自己的部落格了。

建立開發部落格專案

建立空白專案

登入 github 後，先建立一個儲存庫 repository，可以取自己好記的部落格專案名稱，
勾選 private 選項,避免曝露預期存放金鑰以及撰寫中文件。


建立金鑰

要能夠從本地拉取 github 線上的專案，必須先建立有加密的通道才能拉取。
github 提供 HTTPS 以及 SSH 供使用者選擇拉取，這裡採用 SSH。
打開 cmd 輸入以下指令 (指令補充說明)，產生公鑰及私鑰：

預設建立好的 金鑰位置會放在 “C:\Users\{user}\.ssh\“ 的資料夾底下，
分別為 “id_rsa”(私鑰) 以及 “id_rsa.pub” (公鑰)。

設定金鑰

  • 公鑰
    使用文字編輯器打開，將公鑰設定在 github 的帳號
    帳號 → 設定 → SSH and GPG keys
    Title 可以以自己方便記的名稱即可
    Key 的話要依照 open ssh 格式輸入，預設建立出來的公鑰就是依照這個格式，複製貼上即可

  • 私鑰
    Source Tree 選單列 Tools → Options
    SSH Client 改成 OpenSSH
    SSH Key 選擇私鑰 (SSH Client 切換後，預設會選擇 C:\Users\{user}\.ssh\id_rsa)

拉取空白專案

先到 github 專案複製 SSH 的路徑

source tree 切換至 Clone 工具，貼上複製的 SSH 儲存庫路徑後，
如果加密通道設定正確的話，就能正常點擊 Clone 按鈕下載專案

建立 github 個人主頁儲存庫

github 提供以帳號加上固定格式名稱的儲存庫預設為個人主頁
只要依照格式建立的話就能以該名稱建立個人主頁

未來若有添加其他靜態網站的需求，可在新增的儲存庫，透過上傳名稱為 gh-pages 的分支，
github 會以 主頁名稱 + 該專案名稱 為路由建立頁面

發佈 hexo 靜態頁面 至 個人主頁儲存庫

安裝套件之後，至專案內 _config.yml 檔案，
將 deploy 內，個人主頁儲存庫的 SSH 位置設定後保存

以上步驟都完成後，可以在 hexo 專案執行以下指令，進行第一次的個人主頁發佈。

執行完成後，可以至個人主頁儲存庫的 settings → Pages 查看是否已部屬完成
約幾分鐘後就可至 https://{user}.github.io/ 查看個人主頁。

Hexo 專案推送同步部屬個人主頁 (建立 action flow)

我們需要另一組金鑰，在我們 Hexo 部落格專案推送到儲存庫的時候，
自動運行時另一個可以部屬的 SSH 通道，
透過以下指令建立一組新的 金鑰，生成github-deploy-key 以及 github-deploy-key.pub。
預設路徑為： C:\Users\{user} 底下。

接著分別在兩個專案設立私鑰以及公鑰：

  • Hexo 專案
    Name 欄位 輸入 HEXO_DEPLOY_PRI
    Value 欄位 輸入私鑰 github-deploy-key 的內容(begin 和 end 的兩行也需要)

  • 個人主頁
    Title 欄位 輸入 HEXO_DEPLOY_PUB
    Key 欄位 輸入公鑰 github-deploy-key.pub 的內容
    Allow write access 選項需要打勾 (需要權限推送，沒打勾 action 會執行失敗)

個別儲存庫設定好金鑰後，接著就可以設定自動化部屬的腳本：
在 Hexo 專案的 Actions 分頁，點擊 “set up a workflow yourself”
新建一個 main.yml 檔，預設會在(“/.github/workflows/main.yml”路徑底下)

將自動化腳本程式碼覆蓋至 Edit new file 後，
點擊 Start commit，將此修改 commit 至此分支
